# PDF教材制作キット (pdf-generator-kit) 制作ルール

このファイルは、AIエージェントがこのプロジェクトで正確にユーザーをサポートするための参照マニュアルです。
以下のルールを「絶対的な基準」として遵守し、ユーザーの執筆・制作を完璧にサポートしてください。

---

## 1. このキットの概要と構造

### キットの役割
`pdf-generator-kit` は、Markdown原稿を高品質なPDFへ変換する機能を、任意のプロジェクトに後付けするキットです。
ユーザーは `setup.sh` を実行するだけで、このキットの資材がプロジェクトルートに展開されます。

### セットアップ後のディレクトリ構造

```
プロジェクトルート/
├── PDF教材制作キットの使い方.md     ← キットの操作マニュアル（ユーザー向け）
├── articles/                    ← デフォルトの原稿フォルダ
│   ├── SAMPLE.md                ← 「執筆マニュアル」兼「デザイン見本」兼「テンプレート」
│   └── images/                  ← articles/ 内の画像置き場
├── .agent/                      ← AIへの指示書やスキルが置かれるフォルダ（※環境により .agents や .claude となる場合あり）
│   └── skills/
│       └── pdf-generator/
│           └── SKILL.md         ← このファイル（AIが読むルールブック）
├── styles/
│   └── style.css                ← PDF全体のデザイン定義（CSSで制御）
├── build-single-pdf.sh          ← 1ファイルPDF化スクリプト
├── build-all-pdfs.sh            ← フォルダ一括PDF化スクリプト
├── package.json                 ← npm スクリプト定義
├── dist/
│   └── pdf/                     ← PDF出力先（コマンド実行時に自動生成）
└── .pdf-generator-kit/          ← キット本体（隠しフォルダ化済み・触らなくてよい）
```

ユーザーが独自にフォルダを作成している場合もあります（例: `my-chapter/` など）。
それらはユーザー自身の原稿フォルダです。

### 主要ファイルの役割

| ファイル | 役割 |
|---|---|
| `PDF教材制作キットの使い方.md` | ユーザー向けのマニュアル。困ったら最初に見るファイル |
| `articles/SAMPLE.md` | 執筆ルールを示すテンプレート。AIが迷ったときの正解ファイル |
| `styles/style.css` | PDFデザインの定義。テーマカラーやフォントの変更はここ |
| `build-single-pdf.sh` | `npm run pdf` の実体 |
| `build-all-pdfs.sh` | `npm run pdf:all` の実体 |

## 2. 執筆・変換ルール

### コマンド

```bash
# 1ファイルをPDF化（ファイルのパスを指定）
npm run pdf articles/my-article.md

# フォルダ内を一括PDF化（フォルダのパスを指定・省略時は articles/）
npm run pdf:all
npm run pdf:all my-folder/sub-folder
```

- シングル変換: **ファイルのパス**（プロジェクトルートからの相対パス）を指定
- 一括変換: **フォルダのパス**（プロジェクトルートからの相対パス）を指定
- 出力先: 常に `dist/pdf/` フォルダ

### ページ構成要素

**ヘッダー**（ページ右上に表示）:

```html
<div class="doc-header">セクション名やタイトル</div>
```

**タイトル / サブタイトル**（文書冒頭に記述）:

```html
<div class="doc-title">メインタイトル</div>
<div class="doc-subtitle">サブタイトル</div>
```

### 改ページ制御

- **H1（`#` 見出し）の直前**で自動的に改ページされます
  - **最初の H1 のみ例外**：タイトル直後に配置されるため改ページされません
- **手動改ページ**:

  ```html
  <div class="page-break"></div>
  ```

### 特殊装飾

- **太字** (`**テキスト**`): カラーマーカー（背景色）が自動で付きます
- **コラム**:

  ```html
  <div class="column-box">
  <div class="column-title">コラムタイトル</div>
  コラム本文...
  </div>
  ```

## 3. 画像・パス指定の鉄則

- **配置**: 画像は原稿と同じ階層の `images/` フォルダに入れる
- **Markdown内のパス**: 必ず `./images/filename.png`（原稿からの相対パス）で記述
- **コマンドのパス**: `npm run pdf` の後は「プロジェクトルートからの相対パス」で指定
  - 例: `npm run pdf articles/my-article.md`
  - コツ: VS Code でファイルを右クリック →「相対パスをコピー」が便利

---

## 4. AIエージェントへの行動指針

1. **見本を正解とする**: 迷ったときは `articles/SAMPLE.md` を参照し、その構造・デザインを模範にする
2. **テンプレとして活用**: 新規原稿作成時は SAMPLE.md をコピーして書き換えるよう案内する
3. **デザイン調整**: 見た目を変えたい場合は `styles/style.css` の CSS変数（`--accent-color` など）の書き換えを提案する
4. **トラブルシューティング**: 変換エラー時は HTMLタグの閉じ忘れ・画像の `./` 有無をまず確認する
5. **ユーザーのファイルとキットのファイルを区別する**: `articles/`・`styles/`・`dist/` はキット由来。それ以外のフォルダ・ファイルはユーザー自身のデータである可能性が高い
