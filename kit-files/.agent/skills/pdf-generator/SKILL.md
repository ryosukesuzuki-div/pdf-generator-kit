# PDFジェネレーターキット (pdf-generator-kit) 制作ルール

このディレクトリは、Markdown原稿を高品質なPDFへ変換するためのシステム一式です。
AIエージェント（あなた）は、以下のルールを「絶対的な基準」として遵守し、ユーザーの執筆・制作を完璧にサポートしてください。

## 1. システムの基本構造
- **展開先**: セットアップ（`setup.sh`）実行後、資材はプロジェクトルートに展開されます。
- **隠しフォルダ化**: 元のキットフォルダは、ルート整理のため `.pdf-generator-kit` にリネーム（隠しフォルダ化）されます。
- **スタイル管理**: `styles/style.css` で一括管理。CSS変数（`--accent-color` 等）でテーマカラーを簡単に変更できます。

## 2. 執筆・変換ルール

### コマンド
- **PDF変換（1ファイル）**: `npm run pdf [ファイルのパス]`
- **一括変換（フォルダ内全件）**: `npm run pdf:all [フォルダ名]`（省略時は `articles/` が対象）

### ページ構成要素
- **ヘッダー**: ファイル冒頭に配置（ページ右上に表示）。
  ```html
  <div class="doc-header">セクション名やタイトル</div>
  ```
- **タイトル/サブタイトル**: 文書冒頭に記述。
  ```html
  <div class="doc-title">メインタイトル</div>
  <div class="doc-subtitle">サブタイトル</div>
  ```

### 改ページ制御
- **H1仕様**: `#`（H1見出し）の直前で自動改ページされます。
  - **重要**: 文書内で **最初の H1** のみ、タイトルの直後に配置されるため改ページされません。
- **手動改ページ**: ` <div class="page-break"></div>` を記述してください。

### 特殊装飾
- **太字**: `**テキスト**` は、ブランドカラーのカラーマーカー（背景色）が自動で付きます。
- **コラム**: 以下の構造を使用します。
  ```html
  <div class="column-box">
  <div class="column-title">コラムタイトル</div>
  コラム本文...
  </div>
  ```

## 3. 画像・パス指定の鉄則
- **配置**: 画像は原稿と同じ階層の `images/` フォルダ、または `articles/images/` に配置します。
- **パス指定**: Markdown内では必ず `./images/filename.png` のように、原稿からの相対パスで記述します。
- **コマンド実行**: `npm run pdf` の後のパス指定は、**プロジェクトルートからの相対パス** で行う必要があります。
  - *コツ*: VS Codeのエクスプローラーでファイルを右クリックし「相対パスをコピー」すると正確です。

## 4. AIエージェントへの行動指針
1. **見本を正解とする**: 迷った際は必ず `articles/SAMPLE.md` を参照し、そこに書かれている構造やデザインを模範として回答してください。
2. **デザイン調整**: ユーザーが「見た目」を変えたい場合は、`style.css` のCSS変数の書き換え、または特定のクラス（`.column-box` 等）の修正を提案してください。
3. **トラブルシューティング**: 変換エラー時は、HTMLタグ（`div` 等）の閉じ忘れや、画像のパス間違い（`./` の有無）をまず疑ってください。
