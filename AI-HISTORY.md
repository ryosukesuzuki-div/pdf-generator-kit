# pdf-generator-kit：開発の背景と技術メモ（AI引き継ぎ用）

このファイルは、`pdf-generator-kit` の開発経緯と、AIエージェントがコンテキストを把握するための技術メモです。新しいスレッドで作業を開始する際は、まずこのファイルを読み込んでください。

## 🏹 プロジェクトの目標
特定のプロジェクト（Tech Camp等）に依存しない、汎用的でプロフェッショナルな「後付け型」PDF制作構築キットの完成。

## 🛠 主要な機能と設計のこだわり
1. **後付けセットアップ (`setup.sh`)**
   - 既存プロジェクトのルートにキットフォルダを置き、スクリプトを叩くだけで `.agents` (AIスキル) や `styles`, `scripts` を自動展開する。
2. **生きたマニュアル (`articles/SAMPLE.md`)**
   - サンプルファイル自体が「どう書けばどう見えるか」の執筆マニュアルを兼ねている。
3. **柔軟な改ページ制御**
   - H1での自動改ページに加え、`<div class="page-break"></div>` による任意の場所での改ページをサポート（`style.css` に専用クラスを実装済み）。
4. **画像埋め込みサポート**
   - `articles/images/` フォルダを自動生成。原稿からの相対パス `./images/filename.png` での指定を標準化。

## ⚠️ 技術的なハマりどころと解決策
- **改ページが効かない問題**: Vivliostyleの標準機能だけでなく、CSSに `.page-break { break-before: page; }` を明示的に定義することで解決。
- **パスの混乱**: ユーザーがプロジェクトのどこにいても実行できるよう、`npm run pdf [path]` 形式を採用し、相対パスの指定をマニュアルで手厚く解説した。
- **情報のクリーンアップ**: 「教材制作キット」から `pdf-generator-kit` への名称変更、および絵文字や組織名の完全排除を徹底した。

## 🤖 次のAIへの指示
1.  **コンテキストの復元**: 作業を開始する前に、必ず `./.agents/skills/pdf-generator/SKILL.md` を読み込み、このキットの「正解」の書き方を把握せよ。
2.  **時系列の尊重**: ユーザー向けマニュアル (`このキットの使用方法.md`) は、ユーザーの体験時系列（導入→動作確認→執筆→完成）に沿って構成されている。これを崩さないように配慮せよ。
3.  **スタイルの継承**: デザインの調整が必要な場合は、`styles/style.css` の「変数（root）」から変更を提案することを優先せよ。
